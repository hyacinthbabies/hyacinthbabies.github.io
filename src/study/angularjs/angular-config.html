Chapter 21
    Services for REST
    REST是处理http请求的
    $resource:（引入ngResource模块）,当同支持RESTful的数据模型一起工作时，以用它非常方便地同支持RESTful的服务端数据源进行交互。$scope.productsResource = $resource(baseUrl + ":id", { id: "@id" });
    响应会被一个原型类所包装，并在实例上添加一些有用的方法$save/$remove/$delete
	$resource 实例是异步的:
	        $save()
		  $remove()
		  $delete()
	$resource 集合和实例有两个特殊的属性用来同底层的数据定义进行交互:
              $promise （promise）这个属性是特别用来同 $routeProvider.when() 在resolve时进行连接的
              $resolved 属性在服务器首次响应时会被设置为 true （无论请求是否成功）。
              自定义 $resource 方法:
                       需要向包含修改过的 $http 设置对象的资源类传入第三个参数
                       var User = $resource('/api/users/:userId.json', {
							userId: '@id'
							sendEmail: {
							method: 'POST'
						    },
							allInboxes: {
								method: 'JSONP',
								isArray: true
							}
						});
			 $resource 设置对象:
			     对象中的值，也就是动作，是资源对象中某个方法的名字。它可以包含以下键:
			              method:‘GET’、‘DELETE’、‘JSONP’、‘POST’、‘PUT’
			              url
			              params
			              isArray:如果 isArray 被设置为 true ，那么这个动作返回的对象会以数组的形式返回
