第21章 Ajax和Comet

概念：ajax技术的核心XMLHttpRequest对象(简称XHR),它为向服务器发送请求和解析服务器响应提供了流畅的接口

一、XMLHttpRequest对象

//适用于IE7之前的版本
function createXHR(){
    if (typeof arguments.calle.activeXString != “string”) {
        var versions = [“MSXML2.XMLHttp.6.0”, “MSXML2.XMLHttp.3.0”, “MSXML2.XMLHttp”],i, len;
        for (i = 0, len =  versions.length; i<len; i++) {
            try {
                new ActiveXObject(versions[i]);
                arguments.callee.activeXString = versions[i];
                break;
            } catch (ex) {//跳过}
        }
    }
    return new ActiveXObject(arguments.callee.activeXString);
}

ie7以上及其他浏览器都支持原生XHR对象

var xhr = new XMLHttpRequest();

支持所有版本

//是否存在原生XHR

    if(typeof XMLHttpRequest != "undefined"){
        return new XMLHttpRequest();
    }else if(typeof ActiveXObject != "undefined"){//兼容ie7之前}else{//...}}
//后面直接用xhr
var xhr = createXHR();

1.XHR的用法

方法：启动一个请求以备发送
open(arg1,arg2,agr3)

参数：
arg1:要发送的请求的类型(get、post等)
arg2:请求的URL
arg3:表示是否异步发送请求的布尔值

方法：发送特定的请求
send(arg)

参数：
arg:请求主体发送的数据，如果不需要，传入null

同步

等执行完，响应数据会填充XHR的属性

  • responseText:作为响应主体被返回的文本
  • responseXML:响应"text/xml"或"application/xml"时,包含XML Dom文档
  • status:响应的HTTP状态
  • statusText:HTTP状态的说明

demo 

//同步
xhr.open("get","example.php",false);
xhr.send(null);
if((xhr.status >= 200 && xhr.status<300) || xhr.status == 304){
    alert(xhr.responseText);
}else{
    alert('request is unsuccessful'+xhr.status);
}

异步

XHR的readystate属性：请求/响应过程的当前活动阶段

  • 0:未初始化。尚未调用open()
  • 1:启动。已经调用open(),但未调用send()
  • 2:发送。已经调用send(),但尚未接收到响应
  • 3:接收。已经接收到部分数据
  • 4:完成。已经接收到所有数据，并且可以在客户端使用

demo 

//只要readystate变化，就会触发此函数
xhr.onreadystatechange = function(){
    if(xhr.readyState == 4){
        if((xhr.status >= 200 && xhr.status<300) || xhr.status == 304){
            alert(xhr.responseText);
        }else{
            alert('request is unsuccessful'+xhr.status);
        }
    }
};
//异步
xhr.open("get","example.txt",true);
xhr.send(null);

三、进度事件

概念：W3C定义与客户端服务器通信有关的事件，只针对XHR
事件名称 解释 浏览器支持
loadstart 在接收到响应数据的第一个字节触发
progress 在接收响应期间连续不断地触发
error 在请求发生错误时触发
abort 在因为调用abort()方法而终止连接时触发
load 在接收到完整响应数据时触发 ie8只支持load
loaded 在通信完成或者触发error、abort或load事件后触发 都不支持

1.load事件(火狐支持)

2.progress事件(Mozilla革新)

概念：此事件会在浏览器接收数据期间周期性触发

onprogress事件处理程序会接收一个event对象，其target属性指向XHR对象，它还有其他3个属性
属性名 概念
lengthComputable 表示进度信息是否可用的布尔值
position 表示已经接收的字节数
totalSize 表示根据Content-Length响应头部确定的预期字节数
写法事例： 

xhr.onprogress = function(event){
   var divStatus = document.getElementById('status');
   if(event.lengthComputable){
       divStatus.innerHTML = "received" + event.position +"of" +event.totalSize + "bytes";
   }
}

四、跨源资源共享

基本思想：使用自定义HTTP头部让浏览器与服务器沟通，从而决定请求成功还是失败

1.IE对CORS的实现

概要：ie8引入了XDR对象，与XHR类似，但它能够实现安全可靠的跨域通信

XDR与XHR的区别:

  • cookies不会随请求发送，也不会随响应返回
  • 只能设置请求头部信息中的Content-Type字段
  • 不能访问响应头部信息
  • 只支持GET和POST请求

方法：
open(arg1,arg2)

参数：
arg1:请求的类型
arg2:url

XDR请求是异步执行，请求返回会触发load事件，响应的数据会保存在responseText中,如下所示： 

var xdr = new XDomainRequest();
xdr.onload = function(){
    alert(xdr.responseText);
};
//无法确定状态码，失败会触发error事件
xdr.onerror = function(){
    alert("An error occurred.");
};
xdr.timeout = 1000;
xdr.ontimeout = function(){
    alert("Request took too long");
};
xdr.open("post","http://www.somewhere-else.com/page/*);
//为支持POST请求，XDR对象提供contentType属性，用来表示数据发送格式
//此属性是XDR对象影响头部信息的唯一方式
xdr.contentType = "application/x-www-form-urlencoded";
xdr.send("name1=value1&name2=value2");

2.其他浏览器对CORS的实现

概要：Firefox3.5+、Safari4+、Chrome、IOS版Safari和Android平台的Webkit实现了对CORS的原生支持

通过跨域XHR对象可以访问status和statusText属性，支持同步请求,跨域对象也有限制：

  • 不能使用setRequestHeader()设置自定义头部
  • 不能发送和接收cookie
  • 调用getAllResponseHeaders()方法总会返回空字符串

3.Preflighted Requests(预请求)

场景：使用下列高级选项发送请求时，便会向服务器发送Preflight请求：

  • Origin:与简单的请求相同
  • Access-Control-Request-Method：请求自身使用的方法
  • Access-Control-Request-Headers：(可选)自定义的头部信息，多个头部以逗号分隔
发送请求后，服务器通过在响应中发送如下头部与浏览器沟通：
  • Access-Control-Allow-Origin:与简单的请求相同
  • Access-Control-Allow-Methods：允许的方法，多个方法以逗号分隔
  • Access-Control-Allow-Headers：允许的头部，多个头部以逗号分隔
  • Access-Control-Max-Age：应该将这个Preflight请求缓存多长时间(以秒表示)
支持预请求的浏览器包括：Firefox3.5+、Safari4+、Chrome、ie11

4.带凭证的请求

凭据：cookie、HTTP认证及客户端SSL证明

通过将withCredentials属性设置为true，可以指定某个请求应发送凭据。

支持预请求的浏览器包括：Firefox3.5+、Safari4+、Chrome、ie11

5.跨浏览器的CORS

简单请求：非Preflight和不带凭据的请求。

检测XHR是否支持CORS的方法,主要看withCredentials属性： 

function createCORSRequest(method,url){
    var xhr = new XMLHttpRequest();
    if("withCredentials" in xhr){
       xhr.open(method,url,true);
    }else if(typeof XDomainRequest != "undefined"){
       vxhr = new XDomainRequest();
       xhr.open(method,url);
    }else{
       xhr = null;
    }
    return xhr;
}
var request = createCORSRequest("get","http://www.somewhere-else.com/page/");
if(request){
    request.onload = function(){
       //对request.responseText进行处理
    };
    request.send();
}

3.Comet

概念：更高级的Ajax技术，有人称它服务器推送。它是一种服务器向页面推送数据的技术。

实现Comet的2种方式：长轮循和流

轮循：长轮循和短轮循，所有浏览器都支持轮循，使用XHR对象和setTimeout()就可以实现

HTTP流：关键所有服务器端语言都支持打印到输出缓存然后刷新(将输出缓存中的内容一次性全部发送到客户端)的功能
侦听readystatechange事件及检测readystate的值是否为3，利用XHR对象实现HTTP流。 

function createStreamingClient(url,progress,finished){
    var xhr = new XMLHttpRequest(),received = 0;
    xhr.open("get",url,true);
    xhr.onreadystatechange = function(){
        var result;
        if(xhr.readystate == 3){
           //只取得最新数据并调整计数器
           result = xhr.responseText.substring(received);
           received +=result.length;

           //调用progress回调函数
           progress(result);
        }else if(xhr.readystate == 4){
           finished(xhr.responseText);
        }
    };
    xhr.send(null);
    return xhr;
}
//createStreamingClient()接收三个参数：要连接的URL、在接收到数据时调用的函数以及关闭连接时调用的函数
var client = createStreamingClient("streaming.php",function(data){
       alert("Received:"+data);
    },function(data){
       alert("Done!")
    });

4.服务器发送事件

SSE:是围绕只读Comet交互推出的API或者模式。SSE API用于创建到服务器的单向连接，服务器通过这个连接可以发送任意数量的数据。服务器响应的MIME类型必须是text/event-stream

建立连接 

//浏览器向服务器发起连接,url服务器网址必须与当前网页的网址在同一个网域，协议端口必须相同
var source = new EventSource(url);
EventSource实例对象有一个readyState属性，表明连接所处的状态 source.readyState可以取3个值：
  • 0(EventSource.CONNECTING)，表示连接还未建立，或连接断线
  • 1(EventSource.OPEN),表示连接已经建立，可以接收数据
  • 2(EventSource.CLOSED),表示连接已断，且不会重连

open事件

//连接一旦建立，就会触发open事件，可以定义相应的回调函数
source.onopen = function(event){
    //handle open event
}

message事件

//收到数据就会触发message事件
source.onmessage = function(event) {
  var data = event.data;
  var origin = event.origin;
  var lastEventId = event.lastEventId;
  // handle message
};

参数对象event有如下属性：

  • data:服务器端传回的数据(文本格式)
  • origin:服务器端URL的域名部分，即协议、域名和端口
  • lastEventId:数据的编号，由服务器端发送。如果没有编号，这个属性为空

error事件

//如果发生通信错误（比如连接中断），就会触发error事件。
source.onerror = function(event) {
  // handle error event
};

自定义事件

//服务器可以与浏览器约定自定义事件。这种情况下，发送回来的数据不会触发message事件
source.addEventListener("foo", function(event) {
  var data = event.data;
  var origin = event.origin;
  var lastEventId = event.lastEventId;
  // handle message
}, false);
//上面表示浏览器对foo事件进行监听

close事件

//close方法用于关闭连接
source.close();

5.Web Sockets

概念：新浏览器API，目标是在一个单独的持久连接上提供全双工、双向通信。只有支持Web Socket协议的服务器才能工作。

Web Sockets使用自定义协议，URL模式也不同，未加密连接从http://-->ws://,加密的连接从https://-->wss://

创建Web Sockets

//实例一个webSocket对象并传入要连接的URL
var socket = new WebSocket("ws://www.example.com/server.php");
//发送数据(纯文本，其他格式需要序列化)
socket.send("Hello world!");
//服务器返回数据时(纯文本，需要解析)
socket.onmessage = function(event){
    var data = event.data;
    //处理数据
}
//关闭连接
socket.close()

表示当前状态的readyState属性：

  • WebSocket.OPENING(0):正在建立连接
  • WebSocket.OPEN(1):已经建立连接
  • WebSocket.CLOSING(2):正在关闭连接
  • WebSocket.CLOSE(3):已经关闭连接
其他事件(不支持DOM2级)：
  • open:在成功建立连接时触发
  • error：在发生错误时触发，连接不能连续
  • close:在连接关闭时触发。只有close事件event对象有其他三个属性：wasClean、code、reason
var socket = new WebSocket("ws://www.example.com/server.php");
socket.onopen = function(){//...};
socket.onerror = function(){//...};
socket.onclose = function(event){
    console.log(event.wasClean);//布尔值，表示连接是否已经关闭
    console.log(event.code);//服务器返回的数值状态码
    console.log(event.code);//字符串，包含服务器发回的消息
};
